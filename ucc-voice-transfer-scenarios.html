Contact Flows - Transfer Scenarios

Hungary Pilot - Voice Experience
Assessment: Transfer readiness for the Hungary pilot launch
Definition: A transfer occurs when a traveller calls in and the first receiving agent must hand over the call to a second agent for further assistance. The first agent facilitates the connection to ensure the customer is successfully passed to the right person.
For a transfer to be considered successful, the first agent must pass both the actual call (the customer) and the associated interaction record (linked case or reservation) to the second agent.

2. Cisco → AWS

Other Lang agents → HUNG agents

Call Interaction Metadata

Limitations & Workarounds

  • No common CTI layer - metadata lost between platforms
  • Pega case stays locked to Agent 1
  • Agent 1 must manually close Pega to release case
  • Must verbally pass res ID, PIN, verification in warm transfer
  • Cold transfer: warn customer they'll re-verify

Risks

  • Cold: customer re-identifies and re-verifies
  • Forgot to close Pega = Agent 2 locked out
  • Process-dependent - fails under pressure
  • Warm transfers increase AHT
  • Non-intuitive process slows onboarding
1c · 1i vs 1c · 2i

3. AWS → AWS

HUNG agents → HUNG agents

Call Interaction Agent comms

Limitations & Workarounds

  • No direct transfer supported within AWS
  • No metadata, no case reassignment, no warm transfer
  • Can't transfer past shift end to language line
  • Solution: customer redials or re-queues
  • Agent 1 must finalize Pega before ending call
  • Full ID and verification from scratch
Shift mitigation: Sibiu agents can work past shift end - clear instructions needed.

Risks

  • Customer re-queues or redials - abandonment risk
  • All context lost - re-identify, re-verify, re-explain
  • No agent-to-agent communication possible
  • Pega lock if Agent 1 forgets to close
  • Not a transfer - a failed resolution (high dissatisfaction)
1c · 1i vs 2c · 2i
