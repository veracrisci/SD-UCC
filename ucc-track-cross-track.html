Cross-Track

System-level capabilities that shape how all contacts enter, are understood and are orchestrated across channels

Vera Crisci (Service Design) | June 2026

Track Scope

What this track covers

Cross-track captures capabilities that span multiple channels and cannot be scoped to a single track. These are capabilities that affect how all contacts are handled, regardless of whether they arrive via chat, voice or written channels.

The central design principle here is the Intelligent Front Door (IFD): the unified flow that every support interaction passes through, detecting intent, identifying the traveller and routing to the optimal resolution path before a human is involved. Cross-track capabilities either enable the IFD or depend on it.

SPD focus: These are not separate features to be added later. They are system-level capabilities that determine how contacts enter, how they are understood and how they are orchestrated across channels. Design decisions here affect every track.

Cross-track capabilities

  • Real-time translation (language as a system capability, not a channel feature)
  • Routing logic and channel steering
  • IFD capability mapping and design
  • Agent desktop experience (unified across channels)
  • Reporting and measurement framework

Real-Time Translation

CX Assessment summary

The translation initiative (CCAAS-712) aims to enable agents to assist travellers in multiple languages via AWS Connect's native translation capability. SPD's assessment identified three critical observations:

  • No strategic foundation. No defined rationale for which travellers, markets, languages, sites or conditions. Four stakeholders independently flagged this gap.
  • Framed as a chat feature, not a system capability. The Blueprint positions translation at Stage 03 (Classification). The initiative builds it at Stage 05 (Service). Language mismatch is not channel-specific.
  • No experience metrics defined. Technical thresholds (>95% accuracy, <3s latency) exist but there is no connection to CSAT, FCR or effort score.
EPS Strategic Context

Translation is one of four structural shifts in the EPS operating model (Bain). Decouples language from location. By 2030, translated agents could represent 31-59% of all Accommodations text FTE.

POC risk

Targeted POC site cannot do outbound calls. Translation will be tested where agents already can't fully resolve. Cannot isolate translation quality from pre-existing operational gaps.

Routing & Channel Steering

How contacts are classified, prioritised and directed to the optimal resolution path. Includes skill rationalisation (476 skills in Written), priority policy and smart steering logic.

Affects: All tracks | Status: Design input needed

Agent Desktop (Unified)

North Star Principle A3: voice, chat and messaging unified in one interface. Knowledge, case history and actions integrated. Each track builds toward this target independently, and cross-track ensures convergence.

Affects: All tracks | Status: Future alignment

Measurement & Reporting

Consistent CX measurement across channels. CSAT trigger points, RCR methodology and quality monitoring framework. This ensures the program can measure experience improvement, not just migration completion.

Affects: All tracks | Status: Framework needed
