Voice Experience Journey (Pilot)

End-to-End Voice Experience - Hungary Pilot on AWS Amazon Connect

This journey maps the end-to-end traveler and agent experience for inbound voice calls on the Hungary pilot site running on AWS Amazon Connect with Pega integration. It covers the full journey starting from phone dial-in by the traveler, through IVR authentication, queue routing, connection to a live agent, service moment and post-contact measurement, including key agent flows for transfers, escalations and callbacks. Go-live target: 16 June 2026.

Owner: Vera Crisci - Service Design

Core Inbound Call Flow - Hungary Pilot

The primary voice journey for travellers calling in to the Hungary site. Travellers dial via PSTN or Help Center click-to-call, pass through IVR authentication with Reservation ID and PIN, are classified and prioritized by Genius/VIP/emergency status, then routed to the appropriate queue. Once connected, the agent handles the case using Pega CRM with reservation context auto-loaded.

Stage 01
Contact Entry
Traveller reaches voice channel via PSTN or Help Center.
Traveller experience

Dials the phone number via traditional PSTN or clicks "Call us" from the Help Center on web/app. Hears a generic welcome greeting. The dialed number is owned by Amazon Connect which handles the call via contact flow.

Agent experience

Logs into CTI via Okta authentication. Sets phone status to "Available" and waits to receive call interactions. Has no visibility of incoming callers until a call is assigned from the queue.

Available
  • PSTN dial-in (traditional phone)
  • Help Center click-to-call
  • Welcome greeting prompt
  • Language code lookup by DID
  • Business hours check
  • Kill switch via feature flag
Not yet available
  • AITS voice bot entry (Parloa out of scope)
  • Internet calls / click-to-call from app
  • Multi-vertical entry (accommodation only)
Stage 02
Identification
IVR authenticates traveller via Reservation ID and PIN.
Traveller experience

Learns call may be recorded and provides consent via keypad. Enters reservation ID and PIN using DTMF input. Hears confirmation playback of entered details and confirms or re-enters. Gets up to 3 attempts before transfer to agent.

Agent experience

Has no visibility during IVR. Once traveller is authenticated, the reservation page is auto-loaded and displayed in Pega iFrame. Sees authentication status and caller information when connected.

Available
  • Recording consent prompt + DTMF
  • Reservation ID entry via keypad
  • PIN entry and playback confirmation
  • 3 retry attempts before agent fallback
  • Backend fetch via Lambda/API call (8s timeout)
  • High volume prompt (HVP)
  • Non-authenticated entry path
Not yet available
  • Voice biometrics
  • CLI-based auto-identification
  • Multi-reservation selection in IVR
Stage 03
Classification
Reservation attributes and Genius/VIP/emergency status determined.
Traveller experience

Hears reservation status confirmation ("Your reservation is confirmed/cancelled"). Selects request type via keypad - cancellation, payment, or other support. If reservation not found, is informed and transferred directly to an agent.

Agent experience

Does not interact at this stage. Backend evaluates attributes from customer profile - Genius level, VIP status, emergency caller (check-in within 48h), cancellation policy (free vs fees), and reservation status.

Available
  • Reservation status retrieval and playback
  • Request type selection (cancel/payment/other)
  • Genius VIP detection (GL3 priority)
  • Emergency caller detection (48h check-in)
  • Cancellation policy assessment
  • Fallback to agent on failed lookup
Not yet available
  • AI intent detection from speech
  • Topic selection from Help Center diagnose
  • Queue-aware smart classification
Stage 04
Prioritization
Traveller routed to VIP, Priority, or Standard queue.
Traveller experience

Hears "A representative will be available shortly. Please stay on the line." Waits with hold music. If waiting is long, hears offer to switch to English support. Can confirm via keypad to move to ENG queue or stay on the line for Hungarian.

Agent experience

Receives calls from the queue based on availability and routing rules. VIP/Genius Level 3 callers are prioritized. Cannot see queue depth or influence which caller is assigned next. Receives CCP notification in Pega iFrame.

Available
  • VIP queue (Genius L3 / STL)
  • Priority queue (emergency callers)
  • Standard agent queue
  • English fallback queue
  • Hold music during wait
  • Language switch offer on long wait
Not yet available
  • Estimated wait time announcement
  • Callback offer while in queue
  • Skill-based routing beyond language
Stage 05
Service
Live conversation with agent; case handling via Pega.
Traveller experience

Hears audible beep indicating connection. Is greeted by the agent who already has reservation context. Elaborates on request and provides extra details. May be put on hold if agent needs to consult senior or partner. Confirms acknowledgment of next steps.

Agent experience

Greets traveller and reviews caller info received. Verifies identity if needed. Clarifies intent, reviews reservation details and case history. Uses KM Platform for guidelines. Can put caller on hold, perform outbound calls to seniors or partners, and initiate warm or cold transfers.

Available
  • Reservation auto-loaded in Pega
  • Case history review
  • KM Platform guideline access
  • Put caller on hold with music
  • Quick connect for transfers
  • Outbound calls to partner/senior
  • Warm transfer with context handover
  • Cold (blind) transfer to other verticals
  • Click-to-call from Pega
Not yet available
  • AI-assisted reply suggestions
  • Real-time translation
  • Automated case summary generation
  • Concurrent call handling

Transfers & Escalations
▶
1
Transfer Trigger
Traveller

Is informed a transfer is needed - for language support, a different product team, or senior approval.

Agent

Identifies need based on language barrier, shift ending, senior approval, or vertical mismatch. Reviews KM Platform guideline.

Available
  • Language/shift transfer trigger
  • Senior approval consult trigger
  • Cross-vertical transfer trigger
  • Contact number directory
2
Hold & Connect
Traveller

Hears hold music. If hold exceeds 5 min, agent resumes with update. 120s timeout prompt offers alternatives for language transfers.

Agent

Puts traveller on hold. Uses Quick Connect to reach target - senior line, TL extension, language queue, or external number.

Available
  • Hold with music
  • Quick Connect to internal/external
  • 5-min hold update resumption
  • 120s timeout prompt (language)
3
Language Transfer
Traveller

Hears beep when new agent connects. Context handed over warmly - no need to re-explain. If unavailable within 120s, offered English or wait.

Agent

Provides reservation ID, caller details, request, actions needed, and security status during warm handover. Escalates to TL if line closed.

Available
  • Warm transfer with full context
  • Language queue routing
  • TL escalation if line closed
  • English fallback offer
Not yet available
  • Real-time translation
  • Priority for internal transfers
4
Senior Consult
Traveller

Remains on hold with music. Agent resumes if taking longer than 5 min. Hears outcome once consult concludes.

Agent

Uses Quick Connect to reach senior/TL/CSP. Exchanges info and receives approval or guidance. Returns to traveller with outcome.

Available
  • Senior line Quick Connect
  • TL extension + availability check
  • CSP Hungarian line
  • Pega screen-pop for senior
5
Vertical Transfer
Traveller

Informed request needs different team. Hears hold music during connection. If no answer within 60s, told follow-up within 2 days.

Agent

Initiates cold (blind) transfer via Quick Connect to other vertical (Flights, Cars, Cruise, Taxi, Attractions). Cannot do warm transfer to Cisco agents.

Available
  • Cold transfer to verticals
  • 60s answer timeout
  • 2-day follow-up communication
Not yet available
  • Warm transfer to Cisco agents
  • Cross-platform metadata handover

Partner Outbound Calls
▶
1
Partner Call
Traveller

Informed agent needs to contact the property. Agrees to hold. If hold exceeds 5 min, agent resumes with update. May request callback instead.

Agent

Puts traveller on hold. Uses Quick Connect to dial external partner. If no answer within 45s (6 rings), considers email fallback or Language Line.

Available
  • Quick Connect to external partner
  • 45s (6 ring) answer threshold
  • Language Line for non-English partners
  • Email fallback if no answer
  • DTMF input for partner systems
2
Hold Updates
Traveller

If on hold for 5+ min, hears agent resume with status update. Asked if willing to continue waiting. Can request callback.

Agent

Monitors hold time. After 5 min, resumes call to update. Checks if outcome ready. Offers callback or asks to continue holding.

Available
  • 5-min hold threshold for updates
  • Resume and re-hold workflow
  • Callback offer during long holds
  • Audible cue on reconnect
3
Case Continuation
Traveller

Hears the outcome of partner consultation. Agent communicates what was agreed. Confirms acknowledgment of next steps.

Agent

Resumes original case with full context. Communicates outcome from partner call. Agrees on action points and proceeds to closure.

Available
  • Full context preservation
  • Outcome communication
  • Action point agreement
Disconnection & Callbacks
▶
1
Disconnection
Traveller

Call is interrupted unexpectedly. Waits for agent to call back. If intentionally hung up, no callback occurs.

Agent

CCP status auto-switches to ACW. Assesses intentional vs unintentional disconnect. Looks up registered number in Pega Customer 360.

Available
  • Automatic ACW on disconnect
  • Intentional vs unintentional assessment
  • Pega Customer 360 number lookup
  • Click-to-call from Pega
2
Verification Callback
Traveller

Receives call to verified number. Answering automatically counts as identity verification. Continues original case without repeating details.

Agent

Performs outbound call to verified number. Traveller auto-verified on answer. If no pick-up, withdraws case and follows up via GCT email.

Available
  • Outbound call to verified number
  • Automatic verification on answer
  • Continue with original SC type
  • GCT email fallback if no answer
3
Case Resumption
Traveller

Once reconnected, continues original conversation. Agent has full context from previous interaction. Hears resolution and next steps.

Agent

Resumes original service case with full context. Communicates outcome and next steps. Proceeds to case closure and wrap-up.

Available
  • Full context preservation
  • Outcome communication
  • Case closure workflow
Stage 06
Close & Measurement
Call terminated; interaction logged; ACW and wrap-up.
Traveller experience

Hears confirmation of next steps and timeline. Agent greets and waits for traveller to disconnect. If call drops unexpectedly, may receive a callback from the agent for verification and case continuation.

Agent experience

Terminates call and phone status changes to "Unavailable" (After Call Work) automatically for a set duration. Logs and wraps up the interaction in Pega. If disconnected, initiates callback procedure to verified number. Status returns to "Available" after ACW.

Available
  • Automatic ACW status on call end
  • Interaction logging in Pega
  • Call back after disconnection
  • Click-to-call for outbound callback
  • Automatic verification on callback answer
  • GCT email follow-up if no answer
Not yet available
  • Automated call summary generation
  • CSAT survey post-call
  • Full WFM metrics integration
  • Quality monitoring automation
